        try_files $uri $uri/ =404;
    }
}
Config Files: /etc/nginx/nginx.conf, /etc/nginx/conf.d/

Globbing

‚ñº
Description: Uses wildcards to match file patterns, saving time in file operations.
Syntax: * (any chars), ? (single char), [range] (e.g., [1-3])
Examples:

  • rm file[1-3].log: Deletes file1.log, file2.log, file3.log
  • cp /etc/?a* /backup: Copies files like /etc/passwd

Text Processing

‚ñº
Description: Manipulate and search text for efficient data handling.
cut - Extracts sections from lines
Syntax: cut [options] file
Example: cut -d':' -f1,3 /etc/passwd (username and UID)
Options: -d (delimiter), -f (fields)
sort - Sorts lines
Syntax: sort [options] file
Example: sort -k2 -n file (numeric sort by 2nd field)
Options: -k (field), -n (numeric)
tr - Translates characters
Syntax: tr set1 set2 < file
Example: tr '[:lower:]' '[:upper:]' < file
awk - Processes patterns and fields
Syntax: awk 'pattern {action}' file
Example: awk -F: '{print $1,$3}' /etc/passwd
sed - Stream editor for text transformation
Syntax: sed [options] 'command' file
Example: sed -i 's/error/warning/g' log.txt
Options: -i (in-place), -r (extended regex)
grep - Searches for patterns
Syntax: grep [options] pattern file
Example: grep -ri "error" /var/log
Options: -i (ignore case), -r (recursive), -v (invert), -E (extended regex)
Regex Examples:
  • ^error: Lines starting with "error"
  • log$: Lines ending with "log"
  • [0-9]+: One or more digits
  • grep -E 'error|warn' log (match either)
Pipes & Logic
| - Chains output to input
Example: ls /etc | grep conf
&& - Runs next command if previous succeeds
Example: mkdir dir && cd dir
Redirection
> - Overwrites stdout
Example: ls > out.txt

>> - Appends stdout
Example: ls >> out.txt

2> - Redirects stderr
Example: ls nonexist 2> err.txt

< - Redirects stdin
Example: cat < input.txt
