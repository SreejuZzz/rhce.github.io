1. Linux Foundations

Description: Understand Linux’s essence—its open-source kernel, history, distro families, architecture, and file system hierarchy. This sets the context for RHCSA’s focus on RHEL administration.

Config Files

  • /etc: System-wide configs (e.g., /etc/passwd).
  • /var/log: Logs (e.g., /var/log/messages).
  • /boot: Boot files (e.g., /boot/grub2/grub.cfg).

Example: Run man hier to explore File System Hierarchy Standard (FHS).

Study Tip: No commands; read Red Hat docs or skim Linux history on Wikipedia.

2. Terminal & Shell Basics

Description: The terminal is your command-line interface; the shell interprets commands. Bash is RHEL’s default, but others like Zsh exist.

Commands

  • echo $SHELL: Shows current shell.

    # Syntax: echo $SHELL
# Example: echo $SHELL → /bin/bash

  • chsh: Changes default shell.

    # Syntax: chsh -s /path/to/shell
# Example: chsh -s /bin/zsh (sets Zsh for user).

Config Files

  • /etc/shells: Lists available shells.
  • ~/.bashrc: User-specific Bash settings.
  • ~/.bash_profile: Login shell settings.

Example: Add alias ll='ls -l' to ~/.bashrc for a custom alias.

4. Text Editors

Description: Edit configuration files with lightweight editors like **Nano** (simple) or **Vim** (powerful).

Nano

  • nano: Simple editor for quick edits.

    # Syntax: nano [options] [file]
# Example: nano -l /etc/motd (edit with line numbers).
# Options: -l (show line numbers), -w (no wrap).
# Config File: None; settings via options.

Vim

  • vim: Advanced editor with modes (normal, insert, visual).

    # Syntax: vim [options] [file]
# Example: vim /etc/hosts, press i, add 192.168.1.1 server, save with :wq.
# Options: -R (read-only), +10 (start at line 10).
# Config File: ~/.vimrc (e.g., set number for line numbers).

    Cheatsheet:

    • i: Insert mode.
    • Esc: Normal mode.
    • :w, :q, :wq: Save, quit, save+quit.
    • dd, yy, p: Delete line, copy line, paste.
    • /pattern: Search forward.

Example: Create ~/.vimrc with echo "set number" > ~/.vimrc for line numbers in Vim.

5. Globbing

Description: Uses wildcards to match file patterns, saving time in file operations.

Syntax: * (any chars), ? (single char), [range] (e.g., [1-3]).

Examples

  • ls *.txt: Lists all .txt files.
  • rm file[1-3].log: Deletes file1.log, file2.log, file3.log.
  • cp /etc/?a* /backup: Copies files like /etc/passwd.

Config Files: None.

Finding Files

  • which: Locates executable. 
    # Syntax: which command
# Example: which ls → /usr/bin/ls.
  • whereis: Finds binary, source, man pages. 
    # Syntax: whereis command
# Example: whereis bash → bash: /bin/bash /usr/share/man/man1/bash.1.gz.
  • locate: Fast database search. 
    # Syntax: locate pattern
# Example: updatedb ; locate passwd (finds /etc/passwd).
# Config File: /etc/updatedb.conf (controls locate database).
  • find: Advanced file search. 
    # Syntax: find [path] [criteria]
# Example: find / -type f -name "*.conf" -perm 644 (config files with 644 perms).

Links

  • ln: Creates hard or symbolic links. 
    # Syntax: ln [-s] file link
# Example: ln -s /etc/passwd symlink (symbolic link).

Archiving Tools

  • tar: Archives files. 
    # Syntax: tar [cvf/xvf/tvf] file.tar [files]
# Example: tar czvf archive.tar.gz /dir (create gzip archive).
  • gzip: Compresses files. 
    # Syntax: gzip file
# Example: gzip log.txt → log.txt.gz ; gunzip log.txt.gz.
  • bzip2, xz: Higher compression. 
    # Syntax: bzip2 file ; xz file
# Example: xz log.txt → log.txt.xz.

Text Viewing

  • more: Basic pager (forward only). 
    # Syntax: more file
# Example: more /var/log/messages.
  • less: Advanced pager (scroll back). 
    # Syntax: less file
# Example: less /var/log/messages (search with /).
  • head: Shows first lines. 
    # Syntax: head [-n num] file
# Example: head -n 5 /etc/passwd.
  • tail: Shows last lines. 
    # Syntax: tail [-f/-n num] file
# Example: tail -f /var/log/httpd/access_log (live updates).
  • cat: Displays file content. 
    # Syntax: cat file
# Example: cat /etc/hosts.
  • tac: Displays in reverse. 
    # Syntax: tac file
# Example: tac /etc/passwd.

8. Text Processing

Description: Manipulate and search text for efficient data handling.

Commands

  • cut: Extracts sections from lines. 
    # Syntax: cut [options] file
# Example: cut -d':' -f1,3 /etc/passwd (username and UID).
# Options: -d (delimiter), -f (fields).
  • sort: Sorts lines. 
    # Syntax: sort [options] file
# Example: sort -k2 -n file (numeric sort by 2nd field).
# Options: -k (field), -n (numeric).
  • tr: Translates characters. 
    # Syntax: tr set1 set2 < file
# Example: tr '[:lower:]' '[:upper:]' < file (uppercase).
  • awk: Processes patterns and fields. 
    # Syntax: awk 'pattern {action}' file
# Example: awk -F: '{print $1,$3}' /etc/passwd (username, UID).
  • sed: Stream editor for text transformation. 
    # Syntax: sed [options] 'command' file
# Example: sed -i 's/error/warning/g' log.txt (replace in-place).
# Options: -i (in-place), -r (extended regex).
  • grep: Searches for patterns. 
    # Syntax: grep [options] pattern file
# Example: grep -ri "error" /var/log (recursive, case-insensitive).
# Options: -i (ignore case), -r (recursive), -v (invert), -E (extended regex).
  • Regex Examples (for grep):
    • ^error: Lines starting with "error".
    • log$: Lines ending with "log".
    • [0-9]+: One or more digits.
    • grep -E 'error|warn' log (match either).

Pipes & Redirection

  • | (Pipe): Chains output to input. *Example*: ls /etc | grep conf.
  • && (Logical AND): Runs next command if previous succeeds. *Example*: mkdir dir && cd dir.
  • >: Overwrites stdout. *Example*: ls > out.txt.
  • >>: Appends stdout. *Example*: ls >> out.txt.
  • 2>: Redirects stderr. *Example*: ls nonexist 2> err.txt.
  • <: Redirects stdin. *Example*: cat < input.txt.

11. Root & Access Control

Description: **Root** (UID 0) has full access; manage privilege escalation carefully.

Commands

  • su: Switches user. 
    # Syntax: su [-] [user]
# Example: su - root (full root environment).
  • sudo: Runs commands as another user. 
    # Syntax: sudo command
# Example: sudo systemctl restart httpd.
# Config File: /etc/sudoers, /etc/sudoers.d/ (edit with visudo).
# Example: Add bob ALL=(ALL) ALL to /etc/sudoers.d/bob.

Config Files: /etc/sudoers (sudo rules).

Example: Grant sudo to wheel group: visudo, uncomment %wheel ALL=(ALL) ALL.

21. Troubleshooting

Description: Diagnose and fix system, network, storage, and performance issues.

Troubleshooting Tools

  • journalctl -xe: Checks recent errors/logs.
  • fsck /dev/sda1: Repairs filesystems.
  • nmcli con up eth0: Fixes network connection.
  • free -h: Checks memory usage.
  • iotop: Monitors I/O activity.

Password Reset

Boot with rd.break, mount -o remount,rw /sysroot, chroot /sysroot, passwd, touch /.autorelabel.

Config Files: /var/log/ (logs), /etc/selinux/config (for relabeling).

Example: Fix boot issue: Boot to rescue, dracut -f (rebuild initramfs).

22. Web Servers

Description: Host websites with **Apache (httpd)** or **Nginx**, including virtual hosts.

Apache (httpd)

  • Setup: dnf install httpd ; systemctl enable --now httpd

    Virtual Host Example: Create /etc/httpd/conf.d/vhost.conf:

    <VirtualHost *:80>
    ServerName example.com
    DocumentRoot /var/www/example
    <Directory /var/www/example>
        Require all granted
    </Directory>
</VirtualHost>

    Config Files: /etc/httpd/conf/httpd.conf, /etc/httpd/conf.d/.

Nginx

  • Setup: dnf install nginx ; systemctl enable --now nginx

    Virtual Host Example: Create /etc/nginx/conf.d/vhost.conf:

    server {
    listen 80;
    server_name example.com;
    root /var/www/example;
    location / {
        try_files $uri $uri/ =404;
    }
}

    Config Files: /etc/nginx/nginx.conf, /etc/nginx/conf.d/.

Example: Test Nginx: nginx -t ; systemctl reload nginx.

26. NFS

Description: Share files over the network with **Network File System**.

Server Setup

  • Setup: dnf install nfs-utils ; systemctl enable --now nfs-server.
  • Config: Add to /etc/exports: /share *(rw,sync), then exportfs -a.
  • Config File: /etc/exports.

Client Setup

  • Command: mount server:/share /mnt.
  • Config: Add to /etc/fstab: 192.168.1.10:/share /mnt nfs defaults 0 0.
  • Config File: /etc/fstab.

Example: Show exports: showmount -e 192.168.1.10.
